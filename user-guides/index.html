Running a Vulnerability Scan with Harbor

This guide will walk you through the process of pushing an image to Harbor, running a vulnerability scan, and interpreting the results. We'll use Nginx as our example image.

Prerequisites

  • Harbor is deployed and accessible
  • Docker is installed on your local machine
  • You have admin access to your Harbor instance

Steps

  1. Log in to Harbor

Use the Docker CLI to log in to your Harbor instance:

bash docker login harbor.yourdomain.com

Enter your Harbor credentials when prompted.

  1. Create a New Project

  2. Log in to the Harbor web interface

  3. Click on "New Project"
  4. Name your project (e.g., "nginx-test")
  5. Set the project to "Public" for this example

  6. Click "OK" to create the project

  7. Configure Automatic Scanning and SBOM Generation

  8. In your project, go to the "Configuration" tab

  9. Under "Vulnerability scanning", check "Automatically scan images on push"
  10. Under "SBOM generation", check "Automatically generate SBOM on push"

  11. Click "Save" to apply these settings

  12. Pull the Nginx Image

Pull the latest Nginx image from Docker Hub:

bash docker pull nginx:latest

  1. Tag the Nginx Image for Harbor

Tag the image with your Harbor registry:

bash docker tag nginx:latest harbor.yourdomain.com/nginx-test/nginx:v1

  1. Push the Image to Harbor

Push the tagged image to your Harbor registry:

bash docker push harbor.yourdomain.com/nginx-test/nginx:v1

  1. View Scan Results

  2. In the Harbor web interface, navigate to your "nginx-test" project

  3. Find the nginx image you just pushed
  4. Click on the image name to view details
  5. Go to the "Vulnerabilities" tab to see the scan results

  6. Review the list of vulnerabilities, their severity, and any available fixes

  7. View SBOM

  8. In the image details page, go to the "Artifacts" tab

  9. You should see the SBOM artifact associated with your image

  10. Click on the SBOM to view or download it

  11. Interpret the Results

  12. Pay special attention to "High" and "Critical" vulnerabilities

  13. Consider updating to a newer version of Nginx if significant vulnerabilities are found
  14. Investigate mitigation strategies for any unresolved vulnerabilities
  15. Use the SBOM to understand the components of your image and their licenses

Best Practices

  • Regularly scan your images for new vulnerabilities
  • Use the automatic scanning and SBOM generation features for all projects
  • Implement policies to prevent the deployment of images with critical vulnerabilities
  • Review SBOMs to ensure compliance with license requirements and to identify outdated components

By following these steps, you can effectively use Harbor to scan your container images for vulnerabilities and generate SBOMs, helping to maintain the security and compliance of your deployments.
